# EXP-003: H3 — do loops pay their FLOP cost with the 2026 recipe?

- Hypothesis: H3 (backlog, revised 2026-07-04 after literature scan)
- Date: pre-registered 2026-07-04, before any results; run 2026-07-05 (pod B,
  interleaved with EXP-002-AX; 12/12 jobs rc=0, ~1.4h wall, ~$1.40)
- Status: ADJUDICATED — INSTRUMENT-FAIL on gate 1 (K-gap)

## Design

Three arms, identical data/protocol (rewrite + algo_exec, 3 seeds, lr 4e-4):

| arm | model | steps | role |
|-----|-------|-------|------|
| recipe | V1R-loop4: deep_supervision + loop_randomize (clipped Poisson mean 4) + TBPTT-2 + input injection | 6000 | the hypothesis |
| params | V1-loop1 | 6000 | equal-params reference |
| FM | V1-loop1-fm3 | 19500 | matched TOTAL training FLOPs (13/4 layer-pass ratio incl. supervision overhead) |

## Pre-registered adjudication (in order)

1. **K-gap gate (validity):** V1R-loop4 evaluated at K=4 vs K=1 (automatic diagnostic
   rows, notes=K1diag) must differ by > 5 pts aggregate. If not, the readout blind
   spot (arXiv:2606.24898) voids the run — verdict INSTRUMENT-FAIL, not kill.
2. **Primary margin:** V1R-loop4 beats V1-loop1-fm3 by >= 3.0 pts on the family mean
   computed over difficulty tiers 3-5 (the non-saturated band; EXP-001B-ALGO showed
   tier-1/2 ceiling compression), 3 seeds, mean diff > 2x pooled seed SD.
   All-tier means reported as secondary.
3. **Equal-params check:** V1R-loop4 vs V1-loop1 @ 6000 — replicates/extends the
   EXP-001B param-efficiency finding under the recipe.
4. Inference-FLOP accounting reported for all arms (flops_per_correct column);
   recipe model pays 2.5x per token, so a per-inference-FLOP claim additionally
   requires the accuracy gain to exceed that ratio's break-even on flops/correct.

## Decision mapping

- Pass 1+2 -> H3 SCALE-candidate; V3 (H6) proceeds with the recipe baked in.
- Pass 1, fail 2 with positive trend -> owner promising-but-short policy: one EXTEND
  (steps or seeds), then adjudicate.
- Fail 1 -> fix readout/supervision implementation before spending anything more.
- Pass 1, clear negative on 2 -> H3 killed; V3 proceeds memory-only (H2-gated).

## Results

All-tier accuracy by seed (results.csv, run_ids EXP-003-* / EXP-003-FM-*):

| model | rewrite s0/s1/s2 | algo_exec s0/s1/s2 |
|---|---|---|
| V1R-loop4 (recipe) | .305 / .390 / .600 | .475 / .405 / .385 |
| V1-loop1 (equal params) | .400 / .550 / .440 | .485 / .650 / .625 |
| V1-loop1-fm3 (matched FLOPs) | .410 / .455 / .585 | **.755 / .770 / .840** |

**Gate 1 — K-gap (validity): FAIL.** Same checkpoints evaluated at K=1 vs K=4
(notes=K1diag):

- rewrite: .320/.390/.585 (K=1) vs .305/.390/.600 (K=4) — gaps -1.5/0/+1.5 pts
- algo_exec: .460/.405/.365 (K=1) vs .475/.405/.385 (K=4) — gaps +1.5/0/+2.0 pts

Aggregate K-gap < 1 pt, far below the pre-registered 5-pt threshold. The recipe
model is loop-invariant: it learned to route around its recurrence exactly as
the readout blind spot predicts (arXiv:2606.24898). Gates 2-3 are therefore
void, but for the record: V1R-loop4 also trails plain V1-loop1 at equal params
on algo_exec (-16 pts mean) and the FLOP-matched control dominates everything.

## Verdict

**INSTRUMENT-FAIL, not an H3 kill (per pre-registered decision mapping).**

The per-loop supervision implementation produced a model that ignores its loops,
so this run cannot falsify H3 either way. Suspects, in order: (1) per-loop CE
through the shared normalized readout makes K=1 the path of least resistance
(the exact 2606.24898 mechanism our diagnostic was built for); (2) loss weighting
across loop readouts; (3) randomized-K interacting with TBPTT-2 so late loops
rarely receive gradient. Decision mapping says: fix the readout/supervision
implementation before spending anything more on H3. Candidate fixes from the lit
scan: detached-readout per-loop heads, loop-index conditioning of the readout,
increasing bptt_loops, or LOTUS-style step-aligned targets instead of full-answer
CE at every loop.

Cost note: the failure was cheap ($1.40) and the diagnostic worked as designed —
this is the pre-registration system doing its job.

## EXP-003B (retry, pre-registered 2026-07-05 before results)

Fixes implemented in models/loop_core.py (all three from the suspect list):
loop_step_embed (learned per-step embedding at loop entry), detached auxiliary
readouts (only the final step backprops through the loop chain) with linearly
increasing per-step loss weights, bptt_loops 4. Config:
experiments/configs/exp003b_recipe.yaml (V1R2-loop4, 3 seeds, same protocol).
Controls REUSED from EXP-003/EXP-003-FM (seeded data, identical yaml protocol —
declared here, not post-hoc). Gate unchanged: K-gap > 5 pts aggregate else H3
parked permanently.

Local smoke (2026-07-05): mechanical validation only — trains cleanly, K-sweep
works, weighted loss correct. Efficacy NOT locally validated: the CPU-scale
synthetic depth task (S4 prefix products, 16-32 tokens) showed no depth
separation even between 2L and 8L tf_pp references, so it cannot discriminate
recipes. K-gap efficacy rides on the cloud run.

### EXP-003B results (pod C, 2026-07-05, 3/3 jobs clean)

| family | K=4 (s0/s1/s2) | K=1 same ckpt | K-gap |
|---|---|---|---|
| rewrite | .330/.395/.360 | .290/.375/.340 | +4.0/+2.0/+2.0 |
| algo_exec | .400/.420/.430 | .405/.425/.410 | -0.5/-0.5/+2.0 |

Aggregate K-gap ~ +1.5 pts — above EXP-003's ~0 but far below the 5-pt gate.
The fixes moved the needle slightly (rewrite now shows consistent small
positive gaps) but the model remains essentially loop-invariant. Accuracy also
still trails the EXP-003 loop1 reference and the FM control on both families,
so gates 2-3 would fail even if gate 1 had passed.

### Final verdict: H3 PARKED PERMANENTLY

Timebox exhausted (pre-registered: one retry). Three data points now agree —
H1 kill (vanilla loops lose at matched FLOPs), EXP-003 (recipe loop-invariant),
EXP-003B (fixed recipe still loop-invariant, and worse than its own controls).
At this scale, on this benchmark, with two supervision recipes, loops neither
pay their FLOP cost nor even learn to USE their iterations. The residual
param-efficiency finding from EXP-001B stands. Loops are excluded from V3.
Honest scope note for the report: RD-VLA-style wins live in continuous-output
domains; SAGE is symbolic text — the hardest venue for latent iteration.
