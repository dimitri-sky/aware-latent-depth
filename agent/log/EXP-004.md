# EXP-004: thinking-token economics — what converts inference FLOPs into correct answers best?

- Hypothesis: H0b (explicit CoT beats latent computation at matched inference
  FLOPs) — promoted to arc-2 centerpiece after the arc-1 verdicts.
- Date: pre-registered 2026-07-06, BEFORE any training.
- Status: PRE-REGISTERED

## Question

At matched inference FLOPs per answer, which strategy converts compute into
correct answers best: visible chain-of-thought tokens, a bigger model, or latent
delta-rule memory (V3)? Deliverable: exchange-rate curves (accuracy vs
FLOPs/answer) + crossover points, per family.

## Design changes vs the arc-2 plan proposal (decided now, before results)

1. **CoT budget knob = trace granularity at TRAINING time, not eval-time decode
   truncation.** A full-trace model truncated mid-trace never emits `ANSWER:` and
   scores 0 — the proposal's "vary decode budget on the same checkpoints" is an
   invalid instrument that would fake a steep budget curve. Separate models are
   trained on short/med/long traces instead.
2. **B2-wide is sized to match B2-CoT-LONG, not CoT-med.** Per-answer cost is
   prefill-dominated: measured on the real eval distribution, CoT-med is only
   1.15x B2-direct on algo_exec (CoT-long 1.66x). Matching med would give a ~22M
   model — a contrast inside seed noise. One shared B2-wide config, sized on
   algo_exec; on rule_shift it overpays that family's CoT-long by ~1.27x
   (generous to the bigger-model strategy; noted in G3's interpretation).
3. **Added B2-filler control** (contentless '.' tokens, per-instance
   length-matched to the long trace): separates rationale *content* from *extra
   forward passes* (pause-token effect). Without it G1 is ambiguous.
4. **rule_shift gets 2 CoT budgets** (med, long): a "short" tier would be
   FLOP-indistinguishable from med (see table below).
5. **Optional V3-CoT-long arm** (algo_exec, seeds 0-1, secondary): do latent
   memory and visible CoT stack? Time-gated — runs only if the pod session has
   slack (delta decode without KV cache is slow).

Scope statement (pre-registered): arms are *strategy packages* (training recipe +
inference behavior) at fixed steps/batch/seq (4000 x 32 x 768, lr 4.9e-4).
Training FLOPs are NOT matched across arms (CoT trains on longer sequences;
B2-wide pays more per token) — they are reported, and the claim is scoped to
inference-time economics. Greedy decoding only; no sampling/self-consistency.

## Trace formats (emitters in sage/generators/, EXP-004 revision)

- algo_exec: long = per-op lines `ADD 3 -> 7`; med = running register values;
  short = every 3rd value (first/last always); filler = length-matched dots.
- rule_shift: long = full current-rule table + per-char query resolution
  (`rule:\n a -> b ...\nquery:\n ...`); med = per-char query mapping (the
  pre-EXP-004 trace field); filler = length-matched dots.
- Variants are derived deterministically from already-drawn generator state —
  ZERO extra RNG calls. `tests/test_trace_emitters.py` pins prompt/answer
  byte-identity to pre-EXP-004 golden hashes; if that test fails, the reuse of
  EXP-002 runs below is invalid.
- Training form: `prompt-minus-ANSWER:` + supervised `THINK:\n<trace>\nANSWER: <x>`.
  Eval (CoT mode): model generates trace + answer; ONLY the text after the last
  `ANSWER:` is scored (existing scorers); format failures (no ANSWER within
  budget) score wrong and are logged as a rate. Decode budget per arm = p99 of
  the trained suffix length over the eval split + 16 slack:
  algo_exec short/med/long/filler = 47/71/234/233; rule_shift med/long/filler =
  72/155/153. Every generated token is charged a full forward pass at its
  context length (sage/flops/accounting.generation_flops) — CoT is never free.
- Contamination audit extended: train-side text now includes traces.

## Pre-registered reads (margins fixed now)

Primary metric: tier 3-5 family-mean accuracy (EXP-007 ceiling lesson; 120
instances/run). Margin convention: +3.0 points and mean diff > 2x pooled seed SD.
Ceiling guard: if the two most accurate arms on a family both exceed .95 on
tiers 3-5, the primary read narrows to tier 5 for that family.

- **G1 (does CoT pay?)** per family: best CoT budget vs B2-direct >= +3.0 and
  > 2x pooled SD -> "CoT pays" on that family. Best-of-budgets is the gate;
  every budget's delta is reported (no cherry-picking hidden).
- **G2 (content or compute?)**: B2-CoT-long vs B2-filler-long >= +3.0 -> content.
  If B2-filler-long vs B2-direct >= +3.0 while CoT-long - filler < 3.0 ->
  "compute-not-content" (pause-token effect). Both below margin -> no effect to
  attribute.
- **G3 (tokens or params at matched FLOPs?)**: B2-CoT-long vs B2-wide on
  algo_exec (matched +-10% by construction): |delta| < 3.0 -> tie; else the
  winner is named. On rule_shift B2-wide spends 1.27x MORE than CoT-long: a
  B2-wide loss there is a strong anti-params result; a B2-wide win carries the
  overspend caveat.
- **G4 (latent vs visible)**: V3-direct vs B2-CoT-long (rule_shift: median-V3).
  V3 DOMINATES if V3 >= CoT-long - 3.0 AND V3 fpa(3-5) <= 0.75x CoT-long's.
  CoT DOMINATES if CoT-long >= V3 + 3.0 (it always spends more, so accuracy must
  pay for the tokens; the rate is read off the curve).
- **Curves (primary deliverable)**: per family, x = measured mean FLOPs/answer on
  tiers 3-5 (fpa35, logged in run notes; reused direct arms use the analytic
  table value — flagged), y = tier 3-5 mean accuracy. Crossover points = where
  the interpolated B2-CoT budget curve crosses V3-direct accuracy and the
  B2-wide point. Secondary views: marginal-over-prefill cost, flops/correct,
  all-tier means, format-failure rates.
- **V3-CoT-long (secondary, if run)**: vs V3-direct and vs B2-CoT-long at margin
  3.0, descriptive (stacking read; 2 seeds, no stability gate).

Decision mapping: gates are independent characterizations — there is no
kill/scale verdict here; H0b resolves as the sign of G4 plus the curve shapes.
EXTEND policy applies to near-misses (add seeds before adjudicating a <2xSD
margin). Any deviation is logged at decision time, before results exist.

## Verdict

**H0b RESOLVED in favor of trained visible CoT at 18M on SAGE: rationale
tokens are the best FLOP-to-accuracy converter tested, beating both a
FLOP-matched wider model (which buys nothing) and V3's latent memory (which is
cheaper per answer but loses on accuracy and reliability).** The win is
content-driven (filler control null/negative) and granularity-sensitive
(sparse traces don't pay; step-by-step traces saturate). Honest scope: trace
supervision is task-aligned and in-distribution; decoding is greedy; the FLOP
axis spans only ~1.0-1.7x direct because prefill dominates at these
prompt/answer shapes. V3-CoT-long stacking arm (optional, session B) pending;
it does not gate this verdict.
