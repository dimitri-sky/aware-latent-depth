# EXP-006: attribution 2x2 — is V2's broad win the delta memory or the window?

- Hypothesis served: H2 attribution (post-EXP-002/AX finding that V2's advantage
  is BROAD — +21.8 on algo_exec, a non-memory family — not memory-specific)
- Date: pre-registered 2026-07-05, before any results
- Status: CONFIGURED

## Design

V2 differs from B2 in exactly two ways: gated delta-rule memory layers and
window-128 SWA. Complete the 2x2 on algo_exec; add B2-SWA on the memory families.

| | full attention | SWA-128 |
|---|---|---|
| no delta | B2-6L (EXP-002-AX, exists) | B2-SWA (EXP-006-AX, new) |
| delta | V2-full (EXP-006-AX, new) | V2-delta (EXP-002-AX, exists) |

- EXP-006-AX: B2-SWA + V2-full on algo_exec, 3 seeds (6 jobs).
- EXP-006-CP / EXP-006-SG: B2-SWA on compress and state_guard, 3 seeds each
  (6 jobs; per-family training per the multi-task dilution lesson).
- Protocol identical to EXP-002 (4000 steps x 32 x 768, lr 4.9e-4, same data
  hashes); existing EXP-002/EXP-002-AX runs are the comparison cells — reuse is
  valid because data generation is seeded and the protocol yaml is unchanged.
- V2-full: window 4096 > max_seq_len 1024, so the sliding mask never engages
  (models/blocks.py applies the mask only when t > window); FLOP accounting
  correctly charges min(T, W) = full attention.
- B2-SWA and V2-full are config-only arms (no new model code). Verify distinct
  config_hash vs base models at launch.

## Pre-registered interpretation (accuracy, algo_exec, all-tier mean, 3 seeds)

Let G_attr(x) = mean(x) - mean(B2). Reference points: G(V2) = +21.8.

1. **Delta-dominant**: B2-SWA within ±3.0 of B2 AND V2-full within ±3.0 of V2
   -> memory mechanism is load-bearing; "memory-consistent" upgrades toward
   "memory-driven" (still generic-computation, not in-session-memory, semantics).
2. **Window-dominant**: B2-SWA within ±3.0 of V2 AND V2-full within ±3.0 of B2
   -> locality prior explains the win; V3 core becomes SWA-first; delta layers
   justified only if EXP-006-MEM shows B2-SWA does NOT reproduce V2's
   compress/state_guard gains.
3. **Both/interaction**: anything else -> report the additive split
   G(B2-SWA), G(V2-full) - G(B2-SWA), and interaction residual.

Memory-family check (EXP-006-CP/SG): if B2-SWA >= V2 - 3.0 on compress AND
state_guard family means, the H2 "memory" framing is retired entirely in favor
of the locality-prior framing.

## Pre-registered FLOP expectation

SWA alone should capture most of V2's inference-FLOP discount (attention cost
min(T,W) vs T). Predicted flops/correct ordering: B2 > V2-full > V2 ~ B2-SWA
(modulo accuracy differences in the denominator). If B2-SWA gets the discount
without the accuracy, the efficiency and accuracy claims decouple and are
reported separately.

## Cost

9 GPU-bound-ish jobs (B2-SWA fast ~10 min; V2-full CPU-bound ~45 min) — ~$1.50
interleaved in the Stage-1/2 pod session.

## Results

12/12 runs clean (pod C, 2026-07-05). All-tier means, 3 seeds:

### 2x2 on algo_exec

| | full attention | SWA-128 |
|---|---|---|
| no delta | B2 .747 | B2-SWA .782 |
| delta | V2-full **.975** | V2 .965 |

Decomposition: window effect **+3.5**, delta effect **+22.8**, interaction -4.5.

### B2-SWA on memory families

| family | B2 | B2-SWA | V2 |
|---|---|---|---|
| compress | .184 | .193 (+0.9) | .263 (+7.9) |
| state_guard | .563 | .522 (**-4.1**) | .612 (+4.8) |

SWA alone does NOT reproduce V2's memory-family gains; on state_guard it is
actively worse than full attention.

### FLOP expectation: FALSIFIED (pre-registered prediction was wrong)

flops/correct on algo_exec: B2 1.07e10, B2-SWA 1.00e10, V2-full 8.03e9,
V2 8.00e9. The window buys only ~7% (attention scores are a small cost share at
d512/T1024); V2's ~25% efficiency edge comes from the accuracy denominator —
i.e. it is ALSO delta-driven, not window-driven.

## Verdict

**DELTA-DOMINANT (pre-registered interpretation 1).** B2-SWA within noise of B2
on algo_exec/compress (state_guard slightly negative); V2-full matches or beats
V2 everywhere. The gated delta-rule layers are the load-bearing ingredient for
BOTH the accuracy and the efficiency advantage; the SWA window is a nearly-free
FLOP trim, not a driver. H2's framing upgrades from "memory-consistent" to
"delta-driven" — with the semantic caveat intact: on algo_exec the delta layers
act as a better generic sequence-computation mechanism, not in-session memory.
V3 design consequence: delta layers are the core; window retained for cost;
loops excluded (see EXP-003B).
