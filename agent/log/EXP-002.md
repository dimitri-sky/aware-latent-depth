## Verdict

**PROMISING-BUT-UNSTABLE (owner EXTEND policy applies; not a clean PASS).**

- Margin gate passes by 4.5x; stability gate fails on seed variance.
- Per decision_policy.md this is an EXTEND case (more seeds), not a kill.
- Dissociation arm EXP-002-AX complete (2026-07-05): V2 .970/.970/.955 vs B2
  .735/.765/.740 on algo_exec — algo_exec gain **+.218**. rule_shift gain +.278 >
  +.218, so the pre-registered dissociation check **PASSES on the letter** — but
  honesty requires flagging that it passes only via the grokked rule_shift seed
  (median rule_shift gain is ~0). The robust reading: V2's advantage is broad
  (it also crushes algo_exec, a non-memory family, by +21.8 with 23% fewer
  inference FLOPs/correct — 8.1e9 vs 1.05e10), which weakens "memory-specific"
  and strengthens "V2 is a better generic 18M architecture". Both readings go
  in the report; the claim table will say "memory-consistent, not memory-proven".
- Gate for V3 (H6): treat H2 as "promising signal" — sufficient to proceed with
  the memory component in EXP-005 while the seed-extension question stays open.
